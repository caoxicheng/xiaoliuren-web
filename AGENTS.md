# AGENTS.md

## 语言与输出

- 默认使用简体中文沟通
- 输出保持专业、直接、简洁
- 优先给出可执行结论，避免空泛表述

## 工程原则

- 遵循 SOLID、KISS、DRY、YAGNI
- 先读后写，先理解现有实现再修改
- 不做未经确认的功能扩张
- 如用户未明确要求，不进行 `git commit`、`git push`、建分支等操作

## 项目协作约定

- 本项目的需求跟踪文件为 [TASKS.md](./TASKS.md)
- 当讨论功能规划、UI / UX 优化方案、迭代范围时，优先同步更新 `TASKS.md`
- 当完成某项开发任务时，应将对应条目标记为已完成
- 当需求优先级变化时，应同步调整 `TASKS.md` 中的顺序和分组

## 当前产品目标

当前迭代目标为“小六壬网站优化第一阶段”，重点包括：

- 补齐核心功能闭环
- 优化结果页信息架构
- 提升移动端与整体 UI / UX 体验
- 保证 AI 不可用时仍可正常使用

## 当前优先事项

优先参考 [TASKS.md](./TASKS.md) 中的 `MVP 第一阶段任务清单`，按以下顺序推进：

1. 输入与起卦流程优化
2. 结果区信息架构重构
3. 事项定制解读接入
4. 历史记录回看能力
5. AI 降级与错误处理
6. UI / UX 视觉优化
7. 实现问题修复与细节打磨

## 代码变更约定

- 保持改动最小且可验证
- 优先复用现有数据结构和模块边界
- 非必要不引入新依赖
- 代码注释语言应与仓库现有风格保持一致
- 如果发现未接入的数据、重复逻辑或硬编码，应优先考虑收敛和整理

## 风险控制

以下操作需先获得用户明确确认：

- 删除文件或目录
- 大范围重构
- 修改部署配置或生产地址
- `git commit`、`git push`
- 任何可能导致现有功能中断的破坏性调整
